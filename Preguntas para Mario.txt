Respuestas Varias

- El administrador es el que puede crear usuarios, solo deberiamos dejar el sign in del modelo crudlogin
- Cuando crea un usuario, debe comprobarse si se repite el usuario
- Solo necesitamos la direccion del servidor dentro del app.js
- Al agregar una asignatura, la interfaz se parecerá a la de añadir task
- Para cada usuario la vista cambia, se gestiona con ifs con la condición del rol del usuario
- Si queremos una base de datos con un nombre diferente, debe ponerse en el enlace con /elnombrequequeremos en app.js. 
    Ejemplo: mongodb+srv://ellie:1234@cluster0.goxma4m.mongodb.net/fff?retryWrites=true&w=majority  -> fff sería el nombre de la base de datos usada
